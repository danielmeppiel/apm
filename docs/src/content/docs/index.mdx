## Why APM?

AI agents are only as good as their instructions. Today, teams copy-paste prompts across repos, instructions drift out of sync, and there's no way to version, share, or compose agent context.

**APM fixes this.** It brings the same dependency management patterns you know from npm, pip, and Cargo to the world of AI agent instructions.

## Before & After

**Before APM** — Copy-paste instructions between repos, manually maintain AGENTS.md, no versioning, no sharing.

**After APM** — Declare dependencies, install with one command, compile optimized context, share across teams.

```yaml
# apm.yml — declare your agent's dependencies
name: my-project
version: 1.0.0
dependencies:
  code-review: github/user/code-review-instructions
  testing: github/user/testing-best-practices
  security: github/user/security-guidelines
```

```bash
apm install   # fetch and deploy all instruction packages
apm compile   # optimize into scoped AGENTS.md files
```

## Works With Your Tools

APM is runtime-agnostic. It works with GitHub Copilot, Claude, Cursor, Codex, Gemini, and any agent that reads markdown instructions.
